# 우리편 어드민 사용 매뉴얼 & 튜토리얼 v1

**버전**: v1.0
**최종 업데이트**: 2026-04-16
**대상**: 우리편 운영팀 주니어 (처음 어드민을 사용하는 팀원)
**목적**: 어드민의 모든 메뉴를 자신있게 사용할 수 있도록 돕는 실전 가이드

## 📖 이 매뉴얼을 읽는 방법

1. **처음이라면 순서대로**: 0장 → 1장 → 2장 → 3장 순으로 한 번 완독
2. **특정 작업만 하고 싶다면**: 4장 "자주 쓰는 워크플로우"에서 원하는 시나리오 찾기
3. **뭔가 꼬였다면**: 6장 "트러블슈팅" 먼저 확인
4. **절대 금지 사항**: 5장은 꼭 한 번 읽어주세요. 실수 방지용

## 1. 로그인 & 대시보드

### 1-1. 로그인

1. 브라우저에서 https://wooripen-web.vercel.app/admin/login 접속
2. 이메일·비밀번호 입력 후 "로그인" 클릭
3. 성공하면 자동으로 대시보드(`/admin`)로 이동

**로그인 실패 시 안내 메시지**
> "이메일 또는 비밀번호가 올바르지 않습니다."

→ 오타 재확인 → 그래도 안 되면 CMO에게 비밀번호 리셋 요청

### 1-2. 대시보드 한눈에 보기

대시보드에는 4개의 숫자 카드가 있어요.

| 카드 | 의미 | 보는 법 |
|---|---|---|
| 활성 상품 수 | 지금 사이트에 노출 중인 상품 개수 | 유의미한 변동이 있으면 "상품 관리" 확인 |
| 발행된 팁 수 | 공개 상태의 블로그 글 개수 | 매주 증가해야 정상 (신규 콘텐츠 발행 시) |
| 총 상담 건수 | 누적 상담 신청 (전체) | 전체 리드 규모 지표 |
| 미처리 상담 건수 | 아직 처리 안 된 상담 (pending 상태) | **매일 아침 0에 가까워야 함.** 많으면 즉시 처리 |

> 💡 **운영 원칙**: "미처리 상담 건수"가 5건 이상 쌓이면 팀장에게 알림. 상담은 빠른 응대가 생명입니다.

## 3. 메뉴별 상세 가이드

### 3-1. 📞 상담 관리 (Consultations)

**이 메뉴는 뭐하는 곳?**
사이트에서 고객이 "상담 신청" 버튼을 눌러 제출한 리드(예비 고객 정보)를 모아서 처리하는 곳이에요. **우리의 가장 중요한 수익 파이프라인**입니다.

**화면 구성**
- 상단: 상태 필터 버튼 4개 (전체 / 대기 / 연락완료 / 전환완료 / 실패)
- 테이블: 이름, 전화번호, 상담 상품, 사업 구분, 메모, 상태, 생성일시

**기본 사용법**

1. **새 리드가 들어왔다면**
   - 상태가 "대기(pending)"로 표시됨
   - 전화번호 확인 → 고객에게 전화
   - 연락 후 상태를 "연락완료(contacted)"로 변경 (드롭다운 클릭)
   - 계약이 성사되면 "전환완료(converted)"
   - 연락 불가·거절이면 "실패(failed)"

2. **메모 칸 활용**
   - 고객 요구사항·특이사항을 짧게 기록
   - 예: "카페 오픈 4/25, CCTV 4ch 우선 문의"

3. **상태 변경 즉시 저장됨**
   - 별도 저장 버튼 없음. 드롭다운 클릭 → 선택 → 끝

**⚠️ 유의사항**
- 상담 리드는 **삭제하지 마세요.** 데이터 분석용으로 전부 보관.
- 테스트용 데이터는 이름에 "테스트"가 들어가 있어요. 실제 리드와 혼동 금지.
- 개인정보(전화번호)가 들어있으니 **화면 공유 시 주의**.

### 3-2. 💬 Q&A 관리

**이 메뉴는 뭐하는 곳?**
사이트 `/qna` 페이지에서 고객이 올린 질문을 확인하고 답변하는 곳이에요.

**화면 구성**
- 상단: 상태 필터 (전체 / 대기 / 답변완료)
- 테이블: 제목, 작성자, 상태, 조회수, 생성일

**답변 달기 튜토리얼**

1. "대기(pending)" 상태의 질문을 클릭
2. 질문 내용·작성자 정보 확인
3. 답변 입력란(textarea)에 답변 작성
4. "저장" 버튼 클릭
5. 상태가 자동으로 "답변완료(answered)"로 변경됨

**답변 작성 가이드**
- ✅ 2~3문장 이내로 핵심만
- ✅ 전문 용어 최소화 (주부도 이해 가능하게)
- ✅ 가격·약정 같은 민감 정보는 "상담 시 안내드립니다"로 처리
- ❌ "네", "아니오"만 쓰지 마세요. 근거 한 줄이라도 추가
- ❌ 전화번호·이메일을 답변에 쓰지 마세요 (스팸 봇 타겟 됨)

**공개/비공개 토글**
- 비공개 질문(자물쇠 표시)은 작성자 본인만 답변을 볼 수 있음
- 공개 질문은 모든 사이트 방문자에게 노출됨
- 민감한 내용은 비공개로 전환하세요

**⚠️ 유의사항**
- 스팸 질문은 삭제 OK
- 하지만 **실제 고객 질문은 절대 삭제하지 마세요.** 부적절하면 비공개로 돌리세요.

### 3-3. 📖 꿀팁 (블로그·Tips)

**이 메뉴는 뭐하는 곳?**
사이트 `/tips` 페이지에 올라가는 블로그 글을 작성·발행하는 곳이에요.

**화면 구성**
- 상단: "새 글 작성" 버튼
- 테이블: 제목, 카테고리, 상태(draft/published), 조회수, 작성일

**새 글 작성 튜토리얼**

1. "새 글 작성" 클릭 → 에디터 화면 열림
2. **제목 입력** (30자 이내 권장)
3. **slug 확인** — 제목 입력 시 자동 생성됨
   - 예: "키오스크 ROI 계산법" → `kiosk-roi-calculation`
   - ⚠️ slug는 URL 경로가 됩니다. 중복 안 되게 유일하게 작성
   - 한글 있으면 영문·숫자·하이픈만 남도록 자동 변환됨
4. **카테고리 선택**: 인터넷·단말기·CCTV·키오스크·가이드·기타
5. **태그 입력** (쉼표로 구분): `카페, 창업, 수수료`
6. **본문 작성** (TipTap 에디터)
   - 제목(H1/H2), 굵게, 기울임, 목록, 인용, 이미지, 링크 등 사용 가능
   - 이미지는 먼저 "미디어 라이브러리"에 업로드 후 URL 복사해서 삽입
7. **요약(excerpt)** 작성 — 카드 목록에서 보이는 2~3줄 요약문
8. **대표 이미지 URL** 입력 — 카드 썸네일용
9. **SEO 정보 입력** (우측 패널)
   - SEO 제목: 50~60자 (구글 검색 결과에 보이는 제목)
   - SEO 설명: 120~160자
10. **임시저장** 또는 **발행** 버튼 클릭

**임시저장 vs 발행 차이**
- 임시저장(draft): 내부만 볼 수 있음. 작성 중이거나 검토 중인 글
- 발행(published): 사이트에 공개됨. 실제 방문자가 볼 수 있음

**⚠️ 유의사항**
- **slug 중복 금지**: 이미 쓴 slug는 못 써요. "kiosk-roi-calculation-2"처럼 변형하세요
- **발행 전 맞춤법 확인**: 꼭 "임시저장" 상태에서 미리보기로 한 번 보세요
- **이미지 저작권**: 무료 이미지 사이트(Unsplash, Pexels) 또는 직접 촬영본만 사용
- **경쟁사 이름 직접 언급 자제**: "타사 대비" 등 비교는 사실 기반만

### 3-4. ❓ FAQ 관리

**이 메뉴는 뭐하는 곳?**
사이트 곳곳에 표시되는 "자주 묻는 질문" 블록의 내용을 관리하는 곳이에요.

**화면 구성**
- 상단: 카테고리 필터 버튼 (각 카테고리별 건수 표시)
- 테이블: 카테고리 배지, 정렬순서, 질문, 답변 미리보기, 공개 토글

**FAQ 추가 방법**

1. "새 FAQ 추가" 버튼 클릭
2. 카테고리 선택 (예: 인터넷, 단말기, 공통 등)
3. 질문(Q) 입력 — 실제 고객이 물어볼 법한 구어체로
4. 답변(A) 입력 — 명확하고 짧게
5. 정렬순서 입력 — 낮은 숫자가 위에 표시됨 (예: 1, 2, 3…)
6. 저장

**답변 작성 꿀팁**
- 질문: "얼마나 걸리나요?" → 답변: "평균 3~5 영업일 소요됩니다."
- 질문: "약정이 어떻게 되나요?" → 답변: "기본 3년 약정이며, 1년·2년 옵션도 안내해드립니다."

**⚠️ 유의사항**
- 같은 내용의 FAQ를 여러 번 만들지 마세요 → 검색 SEO에 안 좋음
- 고객이 자주 묻는 "실제" 질문만 추가하세요. 추측으로 만들지 말기

### 3-5. 📦 상품 관리 (Products)

**이 메뉴는 뭐하는 곳?**
사이트에 노출되는 모든 상품(인터넷·단말기·CCTV·키오스크·렌탈)을 관리하는 곳이에요.

**화면 구성**
- 상단: 카테고리 필터 (전체/internet/terminal/cctv/torder/rental)
- 테이블: 정렬순서, 이름, 카테고리, 통신사, 가격, 공개 토글

**상품 수정 튜토리얼**

1. 수정하고 싶은 상품의 행 클릭 → 수정 모달 열림
2. 상단에 "기본 정보" / "상세 정보" 탭 2개 존재

#### 📋 기본 정보 탭

| 필드 | 설명 | 예시 |
|---|---|---|
| 이름 | 상품명 | "LG U+ 비즈넷 1G" |
| 카테고리 | 상품군 | internet, terminal, cctv, torder, rental |
| 통신사 | 해당 시에만 | LG U+, KT, SKT, 다이렉트 |
| 가격 | 월 렌탈료 또는 단가 (원) | 33000 |
| 원가 | 내부 관리용 (고객 미노출) | 25000 |
| 통신속도 | 인터넷 상품에만 | "1Gbps" |
| 기프트 설명 | 사은품 안내 | "가입 시 현금 15만원" |
| 기능 | 쉼표로 구분 | "고정IP, 세금계산서, 24시간 A/S" |

#### 📋 상세 정보 탭 (상세 페이지용)

| 필드 | 설명 |
|---|---|
| slug ⚠️ | 상세 페이지 URL (/products/{slug}) — 필수 |
| hero_title | 상세 페이지 상단 제목 |
| hero_subtitle | 상단 서브 문구 |
| hero_image | 상단 배경 이미지 URL |
| trust_badges (JSON) | 신뢰 배지 JSON 배열 |
| detail_features (JSON) | 상세 기능 JSON |
| comparison_table (JSON) | 비교표 JSON |
| SEO 제목 / 설명 | 검색 엔진용 |

**⚠️ JSON 필드 편집 시 매우 주의!**
- 중괄호 `{}` 대괄호 `[]` 쉼표 따옴표 `""` 빠짐없이
- 편집 후 저장 시 문법 오류 있으면 경고 alert 뜸
- **JSON 작성이 어려우면 CMO에게 요청하세요. 직접 손대다 꼬이면 페이지가 깨집니다.**

**⚠️ 유의사항**
- 상품 "삭제"는 실제로 지우는 게 아니라 `is_visible=false`로 숨기는 것 (soft delete)
- 나중에 다시 살릴 수 있음
- 가격은 **숫자만** 입력 (예: 33000, 쉼표나 "원" 쓰지 마세요)

### 3-6. 🎁 패키지 관리 (Packages)

**이 메뉴는 뭐하는 곳?**
여러 상품을 묶은 "패키지" 상품(예: 초기창업자 패키지, 사업자 응원 패키지)을 관리하는 곳이에요.

**현재 존재하는 패키지**
- startup: 초기창업자 패키지
- support: 사업자 응원 패키지

**패키지 수정 튜토리얼**

1. 패키지 목록에서 수정할 패키지 클릭 → 편집 폼 열림

#### 기본 정보
- slug (URL 경로)
- 이름
- 대상 산업 / 대상 설명
- hero_title / hero_subtitle / hero_image
- hook_copy (후킹 카피 — 메인 설득 문구)
- 가격대 레이블 (예: "월 12만원대부터")
- 공개 여부 / 정렬순서
- SEO 정보

#### detail_sections (JSON 섹션 블록)
이게 가장 중요해요. 패키지 페이지 중간에 나오는 콘텐츠 블록들이에요.

**지원하는 블록 타입**
- `benefits`: 후킹 포인트 3~4개
- `testimonial`: 고객 후기 카드
- `faq`: 자주 묻는 질문
- `cta`: 마지막 전환 블록
- `hero`: 추가 강조 히어로
- `richtext`: 긴 글 블록

**JSON 예시 (benefits 블록)**
```json
{
  "type": "benefits",
  "data": {
    "title": "왜 우리편 패키지일까?",
    "items": [
      {
        "icon": "solar:phone-rounded-bold-duotone",
        "title": "전화 한 통으로 4종 견적",
        "desc": "설명…"
      }
    ]
  }
}
```

#### 패키지 구성 아이템 (Items)

이 부분이 "패키지에 뭐가 들어있느냐"를 정하는 핵심.

1. "아이템 추가" 클릭
2. 상품 드롭다운에서 해당 상품 선택 (필수)
3. **"강조표시" 체크**: 대표 구성품으로 표시할지 (⭐ 배지)
4. 메모 입력: 카드에 표시되는 설명 (예: "4채널 매장 표준 구성")
5. 정렬순서 입력 (1이 가장 앞)
6. 저장

**▲▼ 버튼으로 순서 변경 가능**

**⚠️ 유의사항**
- 상품 선택 안 한 아이템은 저장 전 자동 제거됨
- detail_sections JSON 직접 편집은 위험. CMO 승인 후 작업 권장
- 패키지는 **삭제보다 비공개 전환** 선호

### 3-7. 🖼️ 미디어 라이브러리 (Media)

**이 메뉴는 뭐하는 곳?**
사이트에 쓸 이미지를 업로드·보관하는 창고예요. 블로그 글, 상품 이미지 등에 사용.

**화면 구성**
- 상단: 파일 업로드 버튼 (다중 파일 가능)
- 그리드: 업로드된 이미지 목록 (썸네일·파일명·크기·치수)

**이미지 업로드 튜토리얼**

1. "파일 선택" 클릭
2. PC에서 이미지 파일 여러 개 선택 (JPG/PNG)
3. 업로드 시작 — **자동으로 WebP로 변환**되고 1200px로 리사이즈됨
4. 완료되면 그리드에 표시됨
5. 마우스를 이미지에 올리면 "URL 복사" 버튼 나타남
6. URL 복사 → 블로그 에디터나 상품 이미지 필드에 붙여넣기

**이미지 파일 가이드**
- 최대 용량: **5MB**
- 권장 크기: 가로 1200px 이상 (그 이상은 자동 축소)
- 포맷: JPG, PNG (업로드 후 WebP로 변환되어 용량 절감)
- 파일명: 영문·숫자로 (한글 파일명은 URL에서 깨질 수 있음)
- Alt 텍스트: SEO·접근성에 중요. 꼭 입력하세요

**⚠️ 유의사항**
- 이 화면에서는 **이미지 삭제·수정이 안 됩니다** (안전 장치)
- 실수로 올린 이미지는 사용 안 하면 됨
- 진짜 삭제가 필요하면 CMO에게 요청

### 3-10. 📊 SEO 설정

**이 메뉴는 뭐하는 곳?**
구글 검색 결과에 나타나는 우리 페이지의 제목·설명을 관리하는 곳이에요.

**페이지별로 따로 설정됨**
- 메인 `/`
- 각 카테고리 `/internet`, `/business/terminal` 등
- 패키지 `/packages/startup`
- 기타

**편집 튜토리얼**

1. 페이지 목록에서 수정할 페이지 클릭
2. 필드 입력:
   - **SEO 제목**: 50~60자 권장 (너무 길면 구글이 잘림)
     - 예: "사업자 인터넷 비교·신청 | 우리편"
   - **SEO 설명(description)**: 120~160자 권장
     - 예: "KT·SKT·LG U+ 사업자 전용 인터넷을 한 번에 비교. 월 22,000원부터. 3분 만에 견적 받기."
   - **OG 이미지 URL**: SNS 공유 시 썸네일로 쓰이는 이미지 URL (1200×630px 권장)
   - **canonical URL**: 특별한 경우 외엔 건드리지 마세요
3. 우측에 구글 검색결과 미리보기가 실시간으로 보임
4. 저장

**⚠️ 유의사항**
- 제목·설명에 **브랜드 "우리편" 꼭 포함**
- 핵심 키워드 앞쪽에 배치 (예: "사업자 인터넷 비교·신청")
- 과장 금지 ("세계 최고", "업계 1위" 등은 나중에 법적 이슈)

## 4. 자주 쓰는 워크플로우 (튜토리얼)

실제 업무에서 자주 하는 작업을 시나리오별로 정리했어요.

### 워크플로우 1: 새 상담 리드 처리

**상황**: 오전에 출근했더니 새 상담이 3건 들어왔어요.

1. `/admin` 대시보드에서 "미처리 상담 건수" 확인 (숫자 증가한 만큼 새 리드)
2. 좌측 메뉴 "상담 관리" 클릭
3. 상태 필터 "대기(pending)"만 선택
4. 리스트에서 위에서부터 하나씩:
   - 고객에게 전화
   - 연결 됐으면 → 요구사항 경청 → 메모 입력 → 상태를 "연락완료"로 변경
   - 연결 안 됐으면 → 메모에 "부재" 기록 → 상태 그대로 두고 오후 재시도
5. 모든 리드 처리 후 "대기" 필터에 0개가 되어야 정상

**목표**: 리드 유입 후 **30분 이내 1차 응대** (업계 평균 대비 빠르게)

---

### 워크플로우 2: 새 블로그 글 작성·발행

**상황**: "키오스크 ROI 계산법" 글을 작성해서 발행하려 해요.

**준비물**
- 본문 원고 (워드/구글독스)
- 대표 이미지 1장 (가로 1200px 이상)
- 본문 내 삽입할 이미지 몇 장

**순서**

1. **이미지 먼저 업로드**
   - 좌측 메뉴 "미디어 라이브러리" → 파일 선택 → 업로드
   - 업로드된 이미지의 URL 복사 (메모장에 임시 저장)

2. **블로그 작성**
   - "꿀팁" 메뉴 → "새 글 작성"
   - 제목, 카테고리, 태그 입력
   - 본문을 TipTap 에디터에 붙여넣기
   - 본문 중간에 이미지 삽입 (아까 복사한 URL 사용)
   - 요약(excerpt) 입력
   - 대표 이미지 URL 입력

3. **SEO 입력**
   - SEO 제목: "키오스크 ROI 계산법 | 우리편"
   - SEO 설명: "인건비 절감 + 객단가 상승 + 렌탈비를 고려한 업종별 실질 ROI 계산 방법"
   - 길이 가이드라인 따르기

4. **임시저장 → 미리보기**
   - "임시저장" 클릭
   - 사이트 `/tips/{slug}` 접속 안 됨 (발행 전이라)
   - 대신 어드민에서 미리보기 기능 또는 초안 상태로 확인

5. **최종 검토**
   - 맞춤법 (한글 맞춤법 검사기 활용)
   - 링크 클릭 테스트
   - 이미지 제대로 보이는지

6. **발행(published) 클릭**
   - 사이트 `/tips`에 카드로 즉시 노출
   - `/tips/{slug}` URL로 본문 공개

**시간 소요**: 1~2시간 (본문 작성 시간 포함)

---

### 워크플로우 3: Q&A에 답변 달기

**상황**: "약정 기간이 끝났는데 어떻게 해야 하나요?" 질문이 들어왔어요.

1. 좌측 메뉴 "Q&A 관리"
2. 상태 필터 "대기(pending)"
3. 해당 질문 클릭
4. 답변 작성란에:
   ```
   약정 만료 시점에 자동 연장되지 않고 월 단위 사용으로 전환됩니다.
   만료 한 달 전에 안내 문자를 보내드리니, 재약정·변경 원하시면
   상담 번호 1600-6116으로 연락 주세요.
   ```
5. "저장" 클릭 → 상태 자동 "답변완료"로 변경
6. 공개 상태 확인 (공개일 경우 사이트에 답변 노출됨)

### 워크플로우 4: 홈 배너 통계 숫자 업데이트

**상황**: "누적 설치 건수가 3,500건을 넘었으니 숫자를 바꾸자"

2. "누적 설치" 카드 클릭

### 워크플로우 5: 상품 가격 수정

**상황**: "LG U+ 인터넷 1G 가격이 33,000원에서 35,000원으로 인상됐어요"

**⚠️ 가격 변경 시 체크리스트**

## 5. 절대 금지 사항 (🚨 빨간불)

### 🚨 절대 하면 안 되는 것

1. **상담 리드 삭제** — 데이터 분석·영업팀 추적용. 이상한 리드도 일단 보존
2. **상품 "진짜" 삭제** — 상품은 비공개 전환만 (soft delete). 복구 어려움
3. **어드민 계정 타인과 공유** — 개인정보 유출 위험
4. **Supabase·Vercel 대시보드 접근 요청** — 개발자·CMO 전용. 운영팀 접근 불필요
5. **detail_sections JSON 직접 수정** — JSON 문법 오류 시 페이지 깨짐. CMO 승인 후에만
6. **로그인 상태에서 공공 PC 이탈** — 반드시 로그아웃하고 떠나기
7. **법적 고지문 임의 수정** — 법무 검토 필수

### ⚠️ 하기 전에 꼭 확인해야 할 것

1. **가격 변경** → 다른 관련 페이지(블로그, SEO)도 동시 업데이트
2. **상품 공개/비공개** → 재고·계약 상태 팀과 동기화
3. **GNB 메뉴 변경** → 사이트 전체 네비에 영향. CMO 승인
4. **홈 통계 숫자 변경** → 과장 없이 실제 데이터 기반
5. **외부 스크립트 변경** → CMO 지시 없으면 금지

## 6. 트러블슈팅

### 문제 1: 로그인이 안 돼요

| 증상 | 원인 | 해결 |
|---|---|---|
| "이메일 또는 비밀번호가 올바르지 않습니다" | 오타 | 재확인 |
| 로그인 후 흰 화면 | 세션 이슈 | 시크릿 모드로 재로그인 |
| 계속 로그인 페이지로 돌아감 | 쿠키 차단 | 브라우저 쿠키 허용 확인 |

→ 해결 안 되면 CMO에게 비밀번호 리셋 요청

### 문제 2: 수정 후 사이트에 반영이 안 돼요

| 원인 | 해결 |
|---|---|
| Vercel 캐시 | 1~5분 기다리기 |
| 브라우저 캐시 | Ctrl+F5 (강력 새로고침) |
| 비공개 토글 OFF | 공개 토글 확인 |
| 저장 버튼 안 눌림 | 다시 저장 |

### 문제 3: 이미지 업로드가 실패해요

| 원인 | 해결 |
|---|---|
| 5MB 초과 | 이미지 용량 줄이기 (tinypng.com 활용) |
| 지원 안 하는 포맷 | JPG/PNG로 변환 |
| 네트워크 끊김 | Wi-Fi 재연결 후 재시도 |

### 문제 4: 저장했는데 데이터가 사라졌어요

→ **즉시 CMO에게 보고**. 시간, 어떤 메뉴, 어떤 데이터였는지 기록.

### 문제 5: 알 수 없는 에러 alert가 떠요

1. 스크린샷 촬영
2. 브라우저 개발자도구 → Console 탭 열기 → 빨간 에러 메시지 캡처
3. 무슨 작업 중이었는지 기록
4. CMO·개발팀에게 보고

## 7. 용어 사전

| 용어 | 뜻 |
|---|---|
| 리드(Lead) | 상담 신청 고객 정보 |
| slug | URL 경로 뒤쪽 (예: /tips/`kiosk-guide` 에서 `kiosk-guide`) |
| SEO | 검색 엔진 최적화. 구글 검색 상위 노출용 |
| OG 이미지 | SNS 공유 시 보이는 썸네일 |
| GNB | 사이트 상단 메뉴 (Global Navigation Bar) |
| WebP | 차세대 이미지 포맷. 용량 작음 |
| soft delete | 진짜 삭제 아니고 숨기기 (복구 가능) |
| draft | 임시저장 상태 |
| published | 발행 상태 (사이트 노출) |
| detail_sections | 패키지 페이지 중간 콘텐츠 블록 (JSON) |
| TipTap | 블로그 에디터 이름 |
| iconify | 아이콘 라이브러리 (solar:xxx 형식) |
