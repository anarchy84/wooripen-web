## 0. 한눈에 (TL;DR)

| 항목 | 내용 |
|---|---|
| **사이트 주소** | https://wooripen-web.vercel.app |

### 테스트 완료 기준 (Definition of Done)

팀원 1명당 아래 3가지를 모두 해보고, 모든 항목을 ✅ 또는 ❌ + 스크린샷으로 보고.
1. **전체 페이지 순회**: GNB 메뉴 10개 + 패키지 2개 페이지를 PC·모바일에서 훑는다
2. **상담 신청 플로우**: 메인 → 맞춤추천 → 상담 신청 → 완료 페이지까지 실제로 제출
3. **버그 리포트**: 발견한 이슈를 양식에 맞춰 정리해 제출

## 2. 테스트 환경

### 2-1. 지원 브라우저
- 크롬 최신 (PC·모바일)
- 사파리 최신 (PC·iPhone)
- 삼성 인터넷 (안드로이드)
- ❌ IE11 지원 안 함

### 2-2. 해상도 테스트
- **데스크탑**: 1920x1080, 1440x900
- **태블릿**: 768x1024 (iPad)
- **모바일**: 390x844 (iPhone), 360x800 (갤럭시)

크롬 개발자도구 → 디바이스 모드로 간편 테스트 가능.

### 2-3. 어드민 로그인

1. https://wooripen-web.vercel.app/admin/login 접속
2. 이메일·비번 입력 (대웅에게 문의)
3. 로그인 성공 시 `/admin` 대시보드로 자동 이동

**로그인 안 될 때**
- 이메일 오타 확인
- 비번 대소문자 확인
- 그래도 안 되면 대웅에게 "어드민 비번 리셋 요청"

## 4. 상담 신청 플로우 End-to-End 테스트

### 시나리오 A: 메인 → 상담 신청 (기본)

1. 메인 페이지 접속
2. 하단 상담 폼까지 스크롤
3. 필수 필드 입력
   - 이름: `테스트`
   - 전화: `010-1234-5678`
   - 관심 상품 체크박스 최소 1개
4. "상담 신청" 버튼 클릭
5. **기대 동작**: 완료 페이지(`/consultation-complete` 또는 유사)로 이동
6. 완료 페이지에 "신청이 접수되었습니다" 메시지 + 예상 연락 시간 표시

### 시나리오 B: 맞춤추천 → 상담 신청

1. `/recommend` 접속
2. 업종 선택 (예: 카페)
3. 추천 상품 확인
4. 상담 폼 작성 + 제출
5. 완료 페이지 도달 확인

### 시나리오 C: 패키지 → 상담 신청

1. `/packages/startup` 접속
2. "무료 견적 신청하기" CTA 클릭
3. `/recommend`로 이동 확인
4. 폼 작성 제출

### 어드민에서 확인

상담 제출 후 5분 이내에 어드민에 새 리드가 보여야 함.

1. `/admin/login` 로그인
2. 좌측 메뉴 "상담 관리" 클릭
3. 목록에 방금 제출한 테스트 건 보이는지 확인
4. 클릭해서 상세 정보 정확히 저장됐는지 체크
5. 상태 변경(대기→연락완료) 테스트

**❌ 리드가 안 보이면?**
- DB 저장 실패 가능성 → 대웅에게 즉시 보고 (콘솔 에러 로그 캡처)

## 7. 팀원 FAQ

**Q1. 사이트가 안 열려요.**
→ 먼저 시크릿 모드로 접속해보세요. 그래도 안 되면 Vercel 장애 가능성 있으니 대웅에게 보고.

**Q2. 어드민 비번을 까먹었어요.**
→ 대웅에게 "비번 리셋 요청" 전달. Supabase에서 수동 리셋 가능.

**Q3. 패키지 후기 내용이 어색한데요.**
→ 현재 버전은 임시 더미 데이터. 실제 후기 받아서 어드민에서 교체 예정. 발견한 어색한 문구는 리포트로 남겨주세요.

**Q4. 전화번호 `1600-6116`가 실제 운영되는 번호인가요?**
→ 네, 운영 중인 대표번호. 통화 테스트는 영업 시간(평일 9~18시)에만.

**Q5. 상담 신청을 실제로 제출해도 되나요?**
→ 네, 테스트 가능. 단 신청 시 **이름에 "테스트"**를 꼭 넣어주세요. 운영팀이 실제 리드와 구분할 수 있도록.

**Q6. 내 이메일을 뉴스레터에 실제로 등록되나요?**
→ 현재(v1)는 UI만 동작. 백엔드 연동 전이라 실제 저장 안 됨. 기능 추가 시 공지 예정.

**Q7. 모바일에서 GNB가 이상해요.**
→ 햄버거 메뉴 아이콘 탭 → 슬라이드 열림. 안 되면 스크린샷 리포트.

**Q8. 페이지가 느린데요.**
→ 크롬 개발자도구 Network 탭에서 느린 요청 확인 + Lighthouse 점수 캡처해서 리포트.

**Q9. 수정한 내용이 사이트에 안 보여요.**
→ Vercel 배포 캐시로 1~5분 지연될 수 있음. 5분 후에도 안 보이면 보고.

**Q10. 컨텐츠 내용이 이상한데 직접 수정해도 되나요?**
→ ❌ 직접 수정 금지. 반드시 대웅이나 담당자 통해서 어드민에서 수정. 코드 직접 건드리면 배포 충돌 위험.
